# DCS Lua Runner MCP Server - Distribution Guide

Quick guide for packaging and sharing the DCS Lua Runner MCP server.

## 📦 What to Share

### Complete Package

Share the entire `dcs-lua-runner` folder with these essential files:

```
dcs-lua-runner/
├── src/
│   └── index.ts              # Source code (TypeScript)
├── build/                    # Generated by npm run build
│   └── index.js              # Compiled JavaScript
├── package.json              # Dependencies list
├── package-lock.json         # Exact dependency versions
├── tsconfig.json             # TypeScript configuration
├── README.md                 # Usage documentation
├── INSTALLATION_GUIDE.md     # Setup instructions
└── DISTRIBUTION.md           # This file
```

### Optional Files (Can be excluded)

- `node_modules/` - Users will install via `npm install`
- `.gitignore` - Git-specific, not needed for distribution

## 📤 Distribution Methods

### Method 1: ZIP Archive (Recommended for Most Users)

1. Ensure the project is built:
   ```bash
   cd dcs-lua-runner
   npm run build
   ```

2. Create a ZIP file containing the folder (exclude `node_modules/`)

3. Share the ZIP file with:
   - **File name**: `dcs-lua-runner-mcp-v1.0.zip`
   - **Instructions**: Point users to `INSTALLATION_GUIDE.md`

### Method 2: GitHub Repository (Best for Developers)

1. Create a GitHub repository

2. Push the code:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/yourusername/dcs-lua-runner-mcp.git
   git push -u origin main
   ```

3. Users clone and install:
   ```bash
   git clone https://github.com/yourusername/dcs-lua-runner-mcp.git
   cd dcs-lua-runner-mcp
   npm install
   npm run build
   ```

### Method 3: NPM Package (Advanced)

For publishing to npm registry:

1. Update `package.json` with your details

2. Publish:
   ```bash
   npm publish
   ```

3. Users install globally:
   ```bash
   npm install -g dcs-lua-runner-mcp
   ```

## 👥 User Setup Summary

### For Cline (VS Code) Users

1. **Extract** the `dcs-lua-runner` folder
2. **Install**: `npm install && npm run build`
3. **Create** `dcs_lua_runner_settings.json` with their DCS credentials
4. **Edit** `cline_mcp_settings.json` to add server configuration
5. **Restart** VS Code

**Config location**: `%APPDATA%\Code\User\globalStorage\saoudrizwan.claude-dev\settings\cline_mcp_settings.json`

### For Claude Desktop Users

1. **Extract** the `dcs-lua-runner` folder
2. **Install**: `npm install && npm run build`
3. **Create** `dcs_lua_runner_settings.json` with their DCS credentials
4. **Edit** `claude_desktop_config.json` to add server configuration
5. **Restart** Claude Desktop

**Config location**: 
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`

### For GitHub Copilot Users

⚠️ GitHub Copilot does not currently support MCP servers. Only Cline and Claude Desktop are supported.

## 🔗 Quick Start for Recipients

Include this in your distribution message:

---

### Quick Start

1. **Prerequisites**: Install Node.js from https://nodejs.org/

2. **Extract** the dcs-lua-runner folder

3. **Open terminal** in the folder and run:
   ```bash
   npm install
   npm run build
   ```

4. **Create settings file** with your DCS credentials (see `dcs_lua_runner_settings.json.template`)

5. **Configure your AI client**:
   - **Cline**: Edit `cline_mcp_settings.json`
   - **Claude Desktop**: Edit `claude_desktop_config.json`
   
   See `INSTALLATION_GUIDE.md` for detailed instructions.

6. **Restart** your AI client

7. **Test**: Ask "What's the current mission time in DCS?"
